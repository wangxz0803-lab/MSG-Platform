Internal Sim 完整采样流水线（10 步）

1. RNG 种子

numpy + torch 全局种子，确保可复现

2. TDD 时隙

帧 → 子帧 → slot → UL/DL 判定 tdd_config.py

3. UE 放置

ISD 内均匀采样 + 最小距离约束，关联最近基站为 serving cell

4. LSP 采样

3GPP 38.901 大尺度参数：路损 + 阴影衰落 + K 因子

5. K 小区 TDL

为每个基站独立生成 TDL-C 信道 [T, RB, BS_ant, UE_ant]

6. SNR / SIR / SINR

信号功率 + 干扰功率 + 噪声功率 → 三个指标

7. 信道估计

LS / MMSE + 时频插值，SRS 跳频累积 channel_est/

8. DL 预编码

SVD 分解估计信道 → Rank-1 预编码矢量 w_dl

9. SSB 测量

多波束扫描 → per-cell RSRP/RSRQ/SINR + 最优波束 ssb_measurement.py

10. ChannelSample

所有结果封装为 Pydantic 模型，落盘 .pt 文件

● PDP 功率时延   ● SRS 空域特征向量(IIR 平滑)   ● PMI 预编码 (3GPP Type-I)   ● DFT 波束能量   ● RSRP 功率  

Bridge 核心步骤

  1. 计算干扰上下文（SIR, 协方差特征分解, MUSIC DoA）
  2. PDP：UL 信道功率时延谱 → 64 维
  3. SRS：空域协方差 R_hh 的 top-4 特征向量（IIR 时间平滑）
  4. PMI：3GPP 38.214 Type-I DFT 码本搜索 → 4 个波束矢量
  5. DFT：UL 信道 top-4 能量 DFT 波束
  6. RSRP：per-antenna + beam-domain + per-cell 三种粒度
  7. 8 个 gate 标量：SRS 奇异值权重、SINR、CQI
data/bridge.py → sample_to_features()

FeatureExtractor (nn.Module)

  • 输入：24 字段 feat_dict (torch.Tensor)
  • 归一化：ProtocolNormalizer → [-1, 1] 区间
  • 嵌入：complex → 2×real 交织，real → 线性投影
  • 门控：SINR + 能量 + CQI 三重质量门
  • 输出：tokens [B, 16, 128] + token_mask [B, 16]

归一化基于 PROTOCOL_SPEC 中的 3GPP 物理范围（dB/linear/discrete）

两种数据消费方式

方式 A：使用预提取特征（推荐）

训练平台直接加载 features/shard-*.pt，tokens 已就绪，无需安装 msg-embedding 包。每个 shard 包含 ~1000 个样本的 [N,16,128] 张量。

方式 B：使用原始数据

训练平台安装 msg-embedding 包（或使用 model_def/ 下的独立文件），调用 ChannelDataset + bridge.sample_to_features() 自行提取。灵活度更高。

编码器

  • 4 层 TransformerEncoder
  • 8 头自注意力 + FFN
  • 可选 SNR 条件注入
  • 扰动感知编码（高斯/平移噪声）

Latent 投影

  • Energy-weighted pooling（MLP 范数 softmax）
  • 两层 MLP：128 → 64 → 16
  • L2 归一化到单位超球面
  • LatentAdapter（~1.5K 参数，微调用）

解码器（评估用）

  • 2 层 TransformerDecoder
  • Per-feature 重建头
  • 推理时不激活
  • 评估重建 NMSE 时使用

3. 评估

🚀 EncoderWrapper

将 ChannelMAE 编码路径封装为确定性推理模块

  • 输入：tokens [B,16,128] + token_mask [B,16]
  • 处理：Encoder → MLP → mask-aware energy pooling → latent projection
  • 输出：L2 归一化 latent [B,16]
  • 支持 model_config 参数加载非默认架构
