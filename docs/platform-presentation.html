Data as Asset

信道是无线 AI 决策的基石——波束管理、干扰协调、资源调度，每一项智能能力都建立在对信道的精确理解之上。但信道数据的现状是碎片化的：仿真引擎各自为政，现网数据缺少无干扰真值，模型评估没有可复现的标准基准。ChannelHub 从数据侧破题，构建信道的全生命周期管理与评估平台。

平台统一接入三种异构仿真引擎与现网实采数据，通过可配置的 Bridge 流水线将任意来源的信道标准化为固定维度张量。仿真数据提供理想无干扰真值——这是模型学习抗干扰能力的关键前提，现网数据无法替代；现网数据则用于分布验证与域适应，二者互补构成完整数据闭环。模型导入自动校验兼容性，锁定测试集 + 6 维评估 + 排行榜，让每一次模型迭代的质量提升有据可查。面向未来，平台将向信道数字孪生、跨频段迁移支撑、在线 A/B 评估持续演进。

01

痛点与动机

△

数据匮乏与碎片化

信道实测成本极高，研究者依赖单一仿真器，数据多样性不足。单源训练的模型在跨场景部署时性能骤降——UMa 训练的模型到 HSR 场景准确率可能腰斩。数据不是「够用就行」，而是必须覆盖目标场景的完整分布。

□

流程割裂无法沉淀

三种仿真引擎各自为政：数据格式不同、特征提取不同、存储方式不同。每次实验都是一次性的，数据用完即弃。信道数据无法像图像数据集那样被标准化、版本化、复用——研究成果难以积累和复现。

○

模型上线无法评估风险

模型训完了，能上线吗？没人说得清。缺少锁定的测试基准，每次评估用不同数据，结果不可比。缺少多维度指标，NMSE 低不代表 embedding 质量好。模型商用决策缺乏量化依据，风险不可控。

02

架构总览

ChannelHub 采用「数据工厂 + 评估引擎」分离架构：平台负责多源采集、自定义流水线处理、统一存储管理、多格式导出，以及模型导入、自动兼容性校验、标准化评估——训练过程外置，与任意框架对接。核心目标：让信道数据成为可沉淀、可追溯、可复用的资产，让模型评估有锁定基准和量化依据。

ChannelHub 架构：数据工厂（左）+ 评估引擎（右），数据闭环驱动持续迭代
03

数据源与接入

ChannelHub 通过统一的 DataSource 抽象接口和 SOURCE_REGISTRY 自动注册机制接入多种异构数据源，每种数据源均有不可替代的价值。三种仿真引擎从不同物理建模层次提供互补的信道数据——统计建模覆盖面广、射线追踪精度高、QuaDRiGa 空间一致性强；现网实采和内部数据上传则打通生产环境验证闭环。所有数据源输出统一映射为 ChannelSample 标准合约，对上层管道完全透明。

04

操作流程

CollectWizard 四步向导引导完成完整数据采集流程。内置 6 种场景预设（UMa 64T / UMi 32T / InH 8T / RMa 4T / mmWave 28G / HSR 350km/h）支持一键填充全部参数，右侧 Canvas 实时预览拓扑布局并支持拖拽调整。平台前端共 9 个功能页面：仪表盘、数据集管理、信道浏览、数据采集、数据处理 (Bridge)、任务监控、运行记录、多运行对比、模型管理与评估。

Step 0 · 数据来源

4 张可选卡片，选中后蓝色边框+对勾确认。三种仿真引擎可用，内部数据上传入口已预留。

  • Internal Sim (Python 内置仿真) — 3GPP 38.901 TDL/CDL 统计模型，纯 Python 无外部依赖，CPU 即可大规模生成，配置最灵活（5 种运动模式、3 种 UE 分布、4 种信道估计）
  • Sionna RT (射线追踪) — Sionna 2.0 + PyTorch + CUDA GPU 加速，支持内置场景 (Munich/Etoile) 和自定义 OSM 地图导入，物理精度最高
  • QuaDRiGa Real (MATLAB) — 本地 MATLAB 实时生成，8 种 3GPP 标准场景 (UMa/UMi/RMa/InH × LOS/NLOS)，空间一致性强，业界对标基准
  • 内部数据上传 — 支持 .mat / .npy / .pt 格式（即将上线，前端已预留 UI）

Step 1 · 组网与设备配置

左侧 5 个折叠面板配置所有参数，右侧 Canvas 实时预览拓扑（支持拖拽调整站点/UE 位置）。可直接使用 6 种预设一键填充。

  • 拓扑配置 — 布局 (hexagonal/linear)、站点数 (1~57)、ISD (50~5000m)、扇区数 (1/3)、基站高度；HSR 专用: HyperCell 分组、轨道偏移、车体穿透损耗、列车尺寸
  • 天线阵列 — BS 6 种预设 (64T~2T) + UE 4 种预设 (4ant~1ant)；双极化面板 [N_H, N_V, N_P] 独立配置，XPD 0~30dB
  • 射频参数 — 载波频率 (700MHz~39GHz)、系统带宽 (5~100MHz)、SCS (15/30/60/120kHz)；NR RB 数自动查表计算；BS 发射功率 0~60dBm
  • 终端配置 — UE 数量 (1~10000)、分布 (uniform/clustered/hotspot)、速度 (0~500km/h)、运动模式 (static/linear/random_walk/random_waypoint/track)、采样间隔 (0.5ms~1s)、UE 功率 (-10~33dBm)
  • QuaDRiGa 专用 — 3GPP 场景选择 (8 种)、时间快照数 (1~100)、每分片 UE 数、跳过生成选项

Step 2 · 信道配置

核心信道仿真参数配置，含 SRS 高级参数折叠面板。Sionna RT 数据源会额外显示场景选择 (Munich/Etoile/自定义 OSM)。

  • 链路方向 — UL / DL / both (TDD 双向成对生成)
  • 信道估计模式 — ideal (仿真真值) / ls_linear (LS 线性插值) / ls_mmse (LS+MMSE 降噪) / ls_hop_concat (SRS 跳频拼接)
  • TDD 配比 — 5 种模式: DDDSU / DDSUU / DDDDDDDSUU / DDDSUDDSUU / DSUUD
  • 信道模型 — CDL-A / B / C / D / E (仅 Internal Sim)
  • 采样数量 — 1~100,000；邻区干扰 UE 数 0~20；可开启邻区预编码投影和干扰信道存储
  • SRS 高级参数 — SSB 波束数 (4/8/16)、SRS 组/序列跳频、SRS 周期 (1~2560 slot)、梳齿 K_TC (2/4/8)、C_SRS/B_SRS/n_RRC、最大秩 (1~4) 和秩选择门限

Step 3 · 确认提交

顶部 12~16 个彩色 Tag 速览全局配置（数据源、拓扑、天线、频段、链路、估计模式等），下方双列详情卡片展示完整参数 + 拓扑预览小图。

  • 参数校验 — Pydantic v2 数据合约 + 前端表单校验，50+ 字段自动验证
  • 任务派发 — POST /api/datasets/collect → 线程直派 Worker，无 Redis 依赖
  • 实时进度 — WebSocket 推送进度百分比和 ETA；任务详情页含实时日志查看
  • 结果存储 — ChannelSample .pt 文件 + Parquet manifest + SQLite 索引
  • 后续流程 — 采集完成 → 数据集页面浏览/导出 (HDF5/WebDataset/PyTorch) → Bridge 特征提取 → 模型评估
05

技术图解

平台多小区干扰建模严格遵循 3GPP 38.901 / 38.211 标准，完整实现了 UL/DL 分离的干扰信道生成。UL 干扰由邻区 UE 各自发送独立 SRS (ZC 序列，38.211 §6.4.1.4) 产生；DL 干扰由邻区 BS 发送 CSI-RS (Gold 序列) 在时频碰撞时产生。干扰功率通过路径损耗差异自动缩放，邻区波束通过 SVD 投影精确建模干扰方向性。

多小区干扰建模：服务 BS 信号 + 邻区干扰信号 + 噪声
UL/DL 干扰模型 — 平台实际实现
UL: Y = H_serving · X_SRS_serving + Σₖ H_intf_k · X_SRS_k + n
DL: Y = H_serving · X_CSIRS_serving + Σₖ H_intf_k · X_CSIRS_k + n
UL 干扰: 每个邻区随机激活 0~num_interfering_ues 个 UE，各 UE 生成独立 SRS 导频 (ZC 序列, n_SRS_ID = cell_id % 1024)
DL 干扰: 邻区 BS 发送 CSI-RS (Gold 序列, n_ID = cell_id % 1024)，仅在时频碰撞时产生干扰
干扰功率缩放: |h_scaled| = |h_raw| · √(P_interferer / P_serving)，基于路径损耗差异自动计算
SIR / SINR 精确计算
SNR = P_serving[dBm] - N₀[dBm]
SIR = P_serving[dBm] - 10·log₁₀(Σₖ 10^(P_intf_k/10))
SINR = -10·log₁₀(10^(-SNR/10) + 10^(-SIR/10))
UL/DL 功率偏移: BS 43dBm vs UE 23dBm → UL SNR = DL SNR - 20dB
P_rx = P_tx - PL(d_3D, f_c, scenario) - SF(σ)，SF ~ N(0, σ²)
邻区干扰预编码投影
W_k = SVD(H_own_k)[:, :rank_k] → P_k = W_k · W_k^H → H_proj = P_k · H_intf
邻区 BS 的干扰能量集中在其对自身 UE 的波束方向，而非各向同性
SVD 分解邻区信道 → 取前 rank 个左奇异向量构造投影矩阵 → 投影干扰信道
rank 由门限 0.1 自动选择 (max_rank=4)，apply_interferer_precoding 可关闭
信道估计 — 4 种模式
ideal: 直接使用仿真真值 H_true (无噪声，仅仿真可用)
LS: Ĥ_LS[k,r] = Y[k,r] · X*[k] / (|X[k]|² + ε)
MMSE: Ĥ_MMSE = R_HH · (R_HH + (1/SNR)·I)⁻¹ · Ĥ_LS
Hop-concat: 多次 SRS 跳频估计拼接，覆盖全带宽
MMSE 协方差: R[k,l] = 1 / (1 + j·2π·(k-l)·Δf·τ_rms)，指数 PDP 先验
干扰感知估计流程: 构建导频栅格 → 叠加干扰+噪声 → 调用估计器 → 输出 [T, RB, BS_ant, UE_ant]
路径损耗 — 3GPP 38.901 (4 种场景)
UMa NLOS: PL = 13.54 + 39.08·log₁₀(d_3D) + 20·log₁₀(f_c) - 0.6·(h_UT-1.5)，σ_SF = 6.0 dB
UMi NLOS: PL = 22.4 + 35.3·log₁₀(d_3D) + 21.3·log₁₀(f_c) - 0.3·(h_UT-1.5)，σ_SF = 7.82 dB
UMa/UMi LOS: PL = 28.0 + 22·log₁₀(d_3D) + 20·log₁₀(f_c) (d ≤ d_bp)，σ_SF = 4.0 dB
InF (室内工厂): PL = 31.84 + 21.50·log₁₀(d_3D) + 19.0·log₁₀(f_c)，σ_SF = 7.56 dB
d_3D: 3D 距离 (m)，f_c: 载频 (GHz)，h_UT: UE 高度 (m)
LOS 断点距离: d_bp = 4·h_BS_eff·h_UT_eff·f_c / c
总损耗 = PL + Shadow_fading + train_penetration_loss (HSR 场景)
06

模型评估与排行榜

平台支持模型快速导入、自动兼容性校验、一键评估和排行榜对比。上传 .pt/.pth/.ckpt 模型文件后自动检测 encoder/decoder/latent_proj 参数结构，确认兼容即可在锁定测试集上运行 6 维标准化评估，结果自动进入排行榜按 KNN 一致性降序排列，金银铜奖牌标识前三名。

△

模型上传与校验

拖拽上传 .pt / .pth / .ckpt 模型文件，后端自动加载 checkpoint 并扫描 state_dict 中的关键 key (encoder / decoder / latent_proj / proj_shortcut)。校验通过返回参数量和兼容性状态，不兼容的模型标记警告但仍可保存。支持关联已有训练 run_id 或自动生成上传 ID。

□

一键评估

在模型管理页面点击「评估」按钮即可触发评估 Job。后端加载 checkpoint → 在锁定测试集上提取 embeddings → 计算 6 维指标 → 输出 metrics.json + embeddings.parquet。全程异步执行，WebSocket 实时推送进度，评估完成自动更新排行榜。

○

排行榜机制

所有已评估模型按 KNN 一致性准确率降序排列，前三名自动标注金银铜奖牌。每条记录展示 run_id、标签、兼容性、KNN Acc、NMSE(dB)、CT、TW、评估时间。支持按任意指标排序切换，快速定位最优模型。

KNN
KNN 一致性 (主指标)
K 近邻样本的物理属性 (SINR/link/source) 相似度。排行榜主排序依据。
KNN_acc = mean(1/(1+d_phys(anchor, neighbor)))
CT
连续性 Continuity
高维空间中的邻域关系在低维 embedding 中的保留程度。
CT: 高维 k-NN 在低维中的排名偏移
TW
可信度 Trustworthiness
低维 embedding 中的近邻是否真正来自高维空间的邻域，检测「入侵者」。
TW = 1 - 2·Σ(rank-k) / (nk(2n-3k-1))
NMSE
重构质量 NMSE
归一化均方误差，衡量 decoder 重建信道 Token 的精度。
NMSE = 10·log₁₀(‖x - x̂‖² / ‖x‖²) dB
COS
余弦分离边界
相似对与非相似对 embedding 余弦距离的分离程度，值越大区分性越强。
margin = mean(cos_sim) - mean(cos_dissim)
TAU
Kendall's τ 排序相关
embedding 距离排序与物理坐标距离排序的秩相关系数。
τ = (concordant - discordant) / C(n,2)
SPLIT LOCK

测试集锁定机制

确保不同模型、不同时间的评估结果在完全相同的数据上产生，实现真正可复现的对比。

  • 锁定后 test/val UUID 列表持久化到 split_meta.json，不可更改
  • 新增数据自动分配到 train 集，已锁定的 test/val 不受影响
  • 每次评估记录 split_version + git_sha，版本不匹配时排行榜标记警告
  • 支持三种拆分策略: random (默认) / by_position / by_beam

ChannelHub 的价值

ChannelHub 解决的不是「能不能仿真信道」的问题，而是「如何让信道数据成为可沉淀的核心资产、让模型商用决策有据可依」的系统性问题。

多源异构采集消除数据多样性瓶颈，自定义 Bridge 流水线实现跨源标准化，Pydantic 合约 + 版本化管理让每一条数据可追溯。多格式导出打通任意训练框架，模型导入即自动校验兼容性，6 维评估 + 锁定测试集 + 排行榜让每一版模型的质量有数可依。

从 UMa 城区到 HSR 高铁，从统计仿真到射线追踪，从原始信道到标准化 Embedding——ChannelHub 让信道表征的研发从「一次性实验」进化为「可复现、可迭代、可评估的工程化流程」，为 MSG 模型的商用落地提供数据基座和质量保障。
