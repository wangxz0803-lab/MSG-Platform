01

痛点与动机

△

数据匮乏与碎片化

信道实测成本极高，研究者依赖单一仿真器，数据多样性不足。单源训练的模型在跨场景部署时性能骤降——UMa 训练的模型到 HSR 场景准确率可能腰斩。数据不是「够用就行」，而是必须覆盖目标场景的完整分布。

□

流程割裂无法沉淀

三种仿真引擎各自为政：数据格式不同、特征提取不同、存储方式不同。每次实验都是一次性的，数据用完即弃。信道数据无法像图像数据集那样被标准化、版本化、复用——研究成果难以积累和复现。

○

模型上线无法评估风险

模型训完了，能上线吗？没人说得清。缺少锁定的测试基准，每次评估用不同数据，结果不可比。缺少多维度指标，NMSE 低不代表 embedding 质量好。模型商用决策缺乏量化依据，风险不可控。

操作流程

CollectWizard 四步向导：选择数据源 → 配置组网拓扑 → 设置信道参数 → 确认提交。支持 6 种场景预设一键填充，实时拓扑预览。

选择数据源

三种数据源各有侧重，可按需混合采集。建议先用 Internal Sim 快速生成基线数据，再用 Sionna RT / QuaDRiGa 补充高保真样本。

  • Internal Sim — 零依赖，CPU 即可运行，适合大规模统计实验
  • Sionna RT — GPU 加速射线追踪，适合特定场景精确建模
  • QuaDRiGa — MATLAB 驱动，空间一致性强，适合移动性研究

组网拓扑配置

配置基站布局、UE 分布和移动性模型。支持蜂窝和线性两种拓扑，实时 SVG 预览。

信道参数配置

配置 SNR 范围、子载波数、天线阵列、干扰模型等。支持场景预设一键填充。

  • SNR 范围 — 默认 -5 ~ 30 dB，可自定义步长
  • 天线配置 — BS: 4×4 双极化, UE: 1×2 双极化 (默认)
  • 子载波 — 52 RB × 12 RE = 624 子载波
  • 干扰模型 — 可配置邻区数、干扰 UE 数、SIR 范围
  • 信道估计 — perfect / LS / MMSE 三种模式
  • 场景预设 — UMa 宏站 / UMi 微站 / InF 工厂 / RMa 农村 / HSR 高铁 / Custom

确认提交

检查所有配置参数，确认后提交采集任务。任务进入队列后可实时查看进度。

  • 参数校验 — Pydantic v2 数据合约自动校验所有字段
  • 任务队列 — 线程直派 Worker，无 Redis 依赖
  • 实时进度 — WebSocket 推送进度百分比和 ETA
  • 结果存储 — ChannelSample .pt 文件 + Parquet manifest + SQLite 索引

技术图解

信道仿真涉及干扰建模、信道估计、路损计算等核心技术。以下图解说明关键物理过程和数学推导。

接收信号 = 服务信道 × 信号 + 干扰信道 × 干扰信号 + 噪声
上行干扰模型
y_ul = H_serving · x_ue0 + Σ(H_interferer_i · x_ue_i · √(P_i/P_0)) + n
SIR_ul = P₀ · |H₀|² / Σ(Pᵢ · |Hᵢ|²)
SINR_ul = P₀ · |H₀|² / (Σ(Pᵢ · |Hᵢ|²) + σ²)
信道估计 — LS vs MMSE
LS 估计: Ĥ_LS = Y · X⁻¹ = H + N · X⁻¹
MMSE 估计: Ĥ_MMSE = R_HH · (R_HH + σ²(XX^H)⁻¹)⁻¹ · Ĥ_LS
LS: 简单无偏，高噪声下误差大
MMSE: 利用信道统计信息降噪，需要协方差矩阵 R_HH
Perfect: 直接使用真实信道 (仅仿真)
路径损耗 — 3GPP 38.901
PL_UMa_LOS = 28.0 + 22·log₁₀(d_3D) + 20·log₁₀(f_c)
PL_UMa_NLOS = max(PL_LOS, 13.54 + 39.08·log₁₀(d_3D) + 20·log₁₀(f_c) - 0.6·(h_UT-1.5))
d_3D: 3D 距离 (m), f_c: 载频 (GHz), h_UT: UE 高度 (m)
07

ChannelMAE

ChannelMAE 采用 Masked Autoencoder 架构进行信道表征自监督预训练。随机 mask 部分 Token，编码器学习重建被遮蔽的特征，同时通过对比学习增强 embedding 的区分性。

上路: MAE 重建损失 (L₂)，下路: 对比学习损失 (InfoNCE)。推理时只用 Encoder → Pooling → Embedding
MAE 重建损失
L_recon = (1/|M|) · Σᵢ∈M ‖xᵢ − x̂ᵢ‖²
M: 被 mask 的 Token 索引集合
xᵢ: 原始 Token, x̂ᵢ: 重建 Token
InfoNCE 对比损失
L_NCE = -log(exp(sim(z, z⁺)/τ) / Σⱼ exp(sim(z, zⱼ)/τ))
z: anchor embedding, z⁺: 正样本 (同位置不同时间)
τ: 温度参数, sim: 余弦相似度
总损失
L_total = L_recon + λ · L_NCE
λ: 对比损失权重，默认 0.1

评估体系

6 维标准化评估体系，从连续性、可信度、邻域一致性、重构质量、空间分离度、排序相关 6 个维度全面度量信道 embedding 质量。测试集锁定 + 版本化保证可复现。

CT
连续性 Continuity
相邻时间步的 embedding 距离应小于阈值。衡量时域平滑性，避免跳变。
CT = (1/T) Σ 𝟙[‖eₜ - eₜ₊₁‖ < ε]
TW
可信度 Trustworthiness
embedding 空间的近邻关系是否忠实反映原始特征空间的近邻关系。
TW = 1 - (2/NK) Σ r(i,j) 偏移量
KNN
邻域一致性 KNN Accuracy
K 近邻分类准确率。embedding 是否保留了场景/信道类别信息。
KNN_acc = (1/N) Σ 𝟙[ŷᵢ = yᵢ]
NMSE
重构质量 NMSE
归一化均方误差，衡量 decoder 重建 Token 的精度。
NMSE = ‖x - x̂‖² / ‖x‖² (dB)
COS
余弦分离度 Cosine Separation
不同类别 embedding 间的余弦距离。越大表示区分性越强。
sep = mean(1 - cos(eᵢ, eⱼ)), yᵢ≠yⱼ
TAU
排序相关 Kendall's τ
embedding 距离排序与物理距离排序的一致性。
τ = (concordant - discordant) / C(n,2)
测试集锁定机制
测试集 UUID 列表锁定后不可更改 (lock=True)，确保不同模型在完全相同的数据上评估。
每次评估记录 split_version，版本不匹配时排行榜标记警告。
支持三种拆分策略: random (默认) / by_position / by_beam。

工程挑战

从原型到产品级系统，解决了 8 个关键工程问题。每个问题都有明确的根因分析、解决方案和量化效果。点击展开查看详情。

01
噪声带宽 24.4 dB 偏差 — 整个系统带宽 vs 单 RB 带宽计算噪声，导致 SNR 虚低
→
根因

噪声功率按整个系统带宽 (52 RB × 12 × 30kHz = 18.72 MHz) 计算，而信号功率按单 RB (12 × 30kHz = 360 kHz) 计算。带宽比 = 52，偏差 = 10·log₁₀(52) ≈ 17.2 dB。加上归一化问题合计 24.4 dB。

解决

统一为 per-RB 噪声计算: σ² = 10^(-SNR_dB/10) / N_sc_per_RB

效果
24.4 dB → 0 dB

仿真 BER 与理论曲线完全吻合

02
TDL 功率差 10 万倍 — 干扰信道与服务信道功率归一化不一致
→
根因

服务信道和干扰信道各自独立归一化，导致二者功率尺度差 5 个数量级 (10⁵ 倍)。干扰被严重高估或低估。

解决

统一 per-link 归一化: 先按路径损耗缩放，再整体归一化到目标 SNR/SIR。

效果
10⁵× → 1×

干扰功率与目标 SIR 精确匹配

03
时域相关仅 0.13 — 每快照独立生成导致相邻帧几乎不相关
→
根因

原始实现每个时间快照独立调用信道生成函数，各快照间无连续性。理论上相邻帧的自相关应接近 J₀(2πf_d·Δt) ≈ 0.9+。

解决

引入 Jakes SoS (Sum-of-Sinusoids) 模型，构建连续时间轴，各快照从同一随机过程采样。

效果
0.13 → 0.88

自相关系数从 0.13 提升至 0.88

04
静态模式位置漂移 — 每快照随机位置，本应固定不动
→
根因

mobility_mode='static' 时，每个快照仍然重新随机 UE 位置，导致"静止"的 UE 实际在跳变。

解决

首帧锁定 UE 位置坐标，后续帧复用相同位置，只更新小尺度衰落。

效果
Δpos = 0

静态 UE 位置完全固定

05
RT 回退策略 — 射线追踪失败后生成独立 TDL，与时间序列割裂
→
根因

Sionna RT 在某些位置/角度下射线计算失败 (无路径到达)。原实现回退到独立 TDL 信道，导致时间序列断裂。

解决

RT 失败时复用上一成功快照并施加多普勒频移，保持时间连续性。记录失败率用于质量监控。

效果

时间序列连续性保持，RT 失败率 < 2%

06
QuaDRiGa 零速度崩溃 — 速度=0 导致 MATLAB 插值 Inf
→
根因

QuaDRiGa MATLAB 内部在计算多普勒频移时用 speed 做除数。speed=0 导致 division by zero → Inf → 整个信道矩阵 NaN。

解决

设置 1e-3 km/h 地板值: speed = max(user_speed, 1e-3)。该速度对应 2.78×10⁻⁴ m/s，位移可忽略。

效果
Inf → 0

零 NaN 输出，静态场景正常工作

07
配对 UL/DL 干扰 — 1444 行大重构，5 阶段从单链路演进到双向干扰独立估计
→
演进历程

Stage 1: 单 DL 链路 → Stage 2: UL/DL 独立生成 → Stage 3: 成对但共享干扰 → Stage 4: 独立干扰信道 → Stage 5: 完整 TDD UL/DL 互惠+独立干扰估计。

关键决策

UL 和 DL 共享相同的大尺度参数 (路损、阴影衰落)，但小尺度衰落和干扰信道独立生成，符合 TDD 互惠性。

效果
1444 行

跨 5 个演进阶段的完整重构，覆盖所有 3 个数据源

08
去 Dramatiq 化 — 从 Redis + Dramatiq 迁移到线程直派 Worker
→
根因

原系统使用 Dramatiq + Redis 做任务队列，引入了 Redis 服务依赖。对于单机部署场景，消息队列是过度设计。

解决

用 Python threading + queue 替代，FastAPI 进程内直派 Worker。WebSocket 直推进度，无外部依赖。

效果

零外部依赖部署
消除 Redis 单点故障
启动时间从 ~5s → <1s

可信度保障

三层保障体系确保从数据采集到模型评估的每一步都可追溯、可复现、可信赖。

L1 · DATA CONTRACT

数据合约层

Pydantic v2 数据合约贯穿全链路，确保每一条 ChannelSample 的格式、类型、值域都经过严格校验。

  • ChannelSample schema 定义所有必填/选填字段及类型约束
  • 采集时实时校验: 缺失字段、类型错误、值域越界即刻拒绝
  • 三源数据统一映射到相同 schema，消除格式差异
  • Parquet manifest 元数据索引，支持快速过滤和查询
L2 · QUALITY GATE

质量门控层

自动化质量检查在数据流转的关键节点设卡，不合格数据不会进入下游。

  • Bridge 特征提取前校验信道矩阵维度和数值范围
  • Token 提取后检查 NaN/Inf，异常样本标记并隔离
  • 数据集拆分时校验无泄漏: 训练集和测试集 UUID 无交集
  • 模型上传时自动兼容性检查: 输入维度、输出形状、参数量
L3 · EVAL LOOP

评估闭环层

标准化评估流程确保不同模型、不同时间的评估结果可比较。

  • 测试集锁定: UUID 列表一旦锁定不可更改
  • 版本追踪: 每次评估记录 split_version，版本不匹配时警告
  • 6 维指标全量计算: CT / TW / KNN / NMSE / Cosine / Kendall
  • 排行榜自动排序 + 金银铜奖牌标识，按 KNN 一致性降序

ChannelHub 的价值

ChannelHub 解决的不是「能不能仿真信道」的问题，而是「如何让信道数据成为可沉淀的核心资产、让模型商用决策有据可依」的系统性问题。

多源异构采集消除数据多样性瓶颈，自定义 Bridge 流水线实现跨源标准化，Pydantic 合约 + 版本化管理让每一条数据可追溯。多格式导出打通任意训练框架，模型导入即自动校验兼容性，6 维评估 + 锁定测试集 + 排行榜让每一版模型的质量有数可依。

从 UMa 城区到 HSR 高铁，从统计仿真到射线追踪，从原始信道到标准化 Embedding——ChannelHub 让信道表征的研发从「一次性实验」进化为「可复现、可迭代、可评估的工程化流程」，为 MSG 模型的商用落地提供数据基座和质量保障。
