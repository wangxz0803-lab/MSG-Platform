# F-005: sionna_rt 静止模式位置跳点

## 问题现象 (引自 Codex R2-005)
`mobility_mode=static`, `ue_speed_kmh=0` 下 8 个 snapshot 中位置序列出现多个离散跳点。

## 根因分析
`_generate_one_sample()` 的 RT 重试逻辑 (line 1151) 在 `ue_pos_override is None` 时，
会随机生成新的 UE 位置重试。在 static 模式下 `iter_samples()` 不生成轨迹，
每个 snapshot 的 `ue_pos_override=None`，因此：
1. 每个 snapshot 独立调用 `_place_ues(rng, sites, 1)` 随机生成位置
2. RT 重试时再次随机换位置
两层随机导致同一次 static 实验的 8 个 snapshot 位置各不相同。

## 修复方案
1. 在 `iter_samples()` 中，当 `mobility_mode=static` 时用固定种子生成一个位置，
   所有 snapshot 共享此位置（通过 `ue_pos_override` 传入）
2. RT 重试逻辑：当 `ue_pos_override` 不为 None 时已不换位置（line 1151 条件），
   所以只需确保 static 模式始终传入 override 即可
3. 种子：使用 `self._seed` 保证可复现性

## 预期效果
static 模式下所有 snapshot 位置完全一致，`max_position_delta_m=0`

## 风险点
- 如果固定位置在 RT 中无法求解（零信道），所有 snapshot 都会回退到 TDL
- 这是正确行为：静止模式下应一致失败，不应偷偷换位置

## 回归验证
- `run_mobility_correlation()` → 检查 static UL/DL 的 position 序列一致
