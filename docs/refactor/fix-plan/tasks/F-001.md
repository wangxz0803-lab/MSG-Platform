# F-001: quadriga_real 方向与估计模式未透传

## 问题现象 (引自 Codex R2-001)
请求 `DL` 后，样本仍输出 `link="UL"`；请求 `ideal` 后，样本仍输出 `channel_est_mode="ls_linear"`。

## 相关代码位置
- `quadriga_real.py:125-170` — validate_config 不解析 link/channel_est_mode
- `quadriga_real.py:276-302` — _iter_from_mat 硬编码 link="UL", channel_est_mode="ls_linear"
- `quadriga_real.py:324-337` — describe() 不含 link/channel_est_mode
- 前端 `CollectWizard.tsx:516-535` — 已正确发送 link/channel_est_mode（前端无需改动）

## 根因分析
Python 包装层在两处硬编码：
1. `validate_config()` 完全不解析 `link`/`channel_est_mode`/`pilot_type_*` 参数
2. `_iter_from_mat()` 输出时写死 `link="UL"`, `channel_est_mode="ls_linear"`

前端已正确构造 config_overrides 包含这些字段，但后端忽略了它们。

## 修复方案

### 1. validate_config 解析新参数
解析 `link`、`channel_est_mode`、`pilot_type_ul`、`pilot_type_dl`、`tdd_pattern`、`num_interfering_ues`。

### 2. _iter_from_mat 使用解析值 + TDD 互易
- `link="UL"` → 保持现有行为，用 MATLAB UL 输出
- `link="DL"` → 从 UL 理想信道通过 `H_DL = conj(H_UL^T)` 生成 DL
- `link="BOTH"` → 生成 paired UL+DL 输出
- `channel_est_mode="ideal"` → h_serving_est = h_serving_true
- `channel_est_mode="ls_linear"` → 使用 MATLAB 的 LS 估计输出

### 3. describe() 补充字段

## 物理依据
TDD 互易：同频 TDD 系统中，UL 和 DL 共享相同的传播路径。在窄带假设下 H_DL = conj(H_UL^T)。
平台 internal_sim 和 sionna_rt 已使用相同方案（_interference_estimation.py:408-425）。

## 预期效果
- summarize_phase1 中 quadriga_real DL 样本的 link="DL"
- quadriga_real ideal 样本的 channel_est_mode="ideal"
- NMSE(ideal vs ideal) = 0 或极小

## 风险点
- DL 信道维度需正确：UL [T,RB,BS_rx,UE_tx] → DL [T,RB,BS_tx,UE_rx]
- 当前 quadriga_real 的 BS/UE 天线数不区分 tx/rx，需确认一致性

## 回归验证
- `summarize_phase1()` → 检查 quadriga_real DL/ideal 样本标签
- `analyze_estimated_quality()` → 检查 NMSE
- 上下行都需验证
